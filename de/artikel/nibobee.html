Installieren der NIBObee Bibliothek auf Linux

Ich habe dieses Tutorial ganz einfach aus dem Grund geschrieben, dass ich keine NIBObee Bibliothek Installation Tutorials für Linux gefunden habe.
Das Betriebssystem welches ich hierfür nutze ist Ubuntu 14.04 LTS Trusty Tahr. Falls Sie Debian benutzen, sollten Sie mit diesem Tutorial gut zu Recht kommen.
Falls Sie einen NIBObee Roboter gekauft haben, erhielten Sie diesen höchstwahrscheinlich mit einer CD.
Der einzig relevante Ordner, den Sie hierfür brauchen ist „linux“.

Öffnen Sie das Stammverzeichnis der CD-ROM und sehen Sie sich den Inhalt an.
Sie sehen 5 tar Archive, welche 4 Programme und die NIBObee-Bibliothek enthalten.

Nun haben Sie zwei Möglichkeiten:
- Installation der 4 Programme avrdude, avr-libc, binutils und gcc von den tar Archiven durch das Entpacken und Kompilieren
- Jedoch empfehle ich ein System wie apt-get zu nutzen, nicht nur aus Sicherheitsgründen, sondern auch wegen seiner Benutzerfreundlichkeit. Also geben Sie folgendes ein:

sudo apt-get install avrdude avr-libc binutils gcc

Nachdem Sie das gemacht haben, ist immer noch viel zu tun. Zuerst müssen wir einen Projektordner für unsere NIBObee-Projekte erstellen.
In diesem Beispiel erstellen wir diesen Ordner auf dem Desktop mit:

mkdir ~/Desktop/nibobee-projects

Angenommen dass Sie die CD benutzen, tippen Sie die folgenden Befehle ein, um ins CD-ROM Verzeichnis zu wechseln und dann die nibobeelib tar Datei in das neulich erstellte Verzeichnis zu kopieren:

cd /media/$USER/NIBObee/linux/
cp nibobeelib-20110603.tgz ~/Desktop/nibobee-projects/

Nun können wir zurück in den Projektordner und das .tgz Archiv extrahieren:

cd ~/Desktop/nibobee-projects
tar xvf $(ls | grep nibobeelib)

Jetzt hängen wir an einem neuen Problem fest ohne es zu realisieren.
Die Macher der CD haben das include Verzeichnis für Linux anders gebündelt als für Windows.
Das include Verzeichnis ist tatsächlich existent! Jedoch ist es im src Ordner vorhanden und heisst nibobee anstatt include.
Keine Sorge, dieses Problem wird in ein paar Momenten gelöst.

Wenn Sie nun das Verzeichnis prüfen, finden Sie die Ordner doc, hex, lib und src. Und die .tgz Datei.

  • doc enthält die Dokumentation der verschiedenen Funktionen aus der NIBObee-Bibliothek. Wenn Sie die NIBObee-Bibliothek genauer lernen möchten, sollten sie diese Ordner definitiv ansehen!
  • hex enthält interessante, vorkompilierte .hex Dateien, die Sie nur noch auf den Roboter übertragen müssen.
  • lib enthält die Bibliotheksdateien.
  • src enthält den Quelltext, die include Dateien, Tutorial Programme und viele mehr.

So habe ich die NIBObee-Bibliothek installiert:

sudo mkdir /opt/nibobeelib/
sudo mv doc hex lib src /opt/nibobeelib/

Jetzt haben Sie alle Dateien die Sie brauchen, gut organisiert im /opt/ Ordner Ihrer Linux Distribution.
Lassen Sie uns ein Programm aus dem /opt/nibobeelib/ Ordner kopieren und in das Verzeichnis wechseln:

cp -r /opt/nibobeelib/src/tutorial/program1 . cd program1

Versuchen Sie das Programm mit make zu kompilieren.
Was ist gerade passiert? Die Konsole zeigt uns nun:

Makefile:13: ../../config-m16-15.mk: No such file or directory
Makefile:14: ../../version.mk: No such file or directory

Keine Sorge. Das ist völlig normal und ich werde jetzt erklären was soeben geschehen ist.
Der Makefile in diesem Verzeichnis benutzt relative Pfade anstelle von absoluten Pfaden.
Wie Sie sehen können, tritt das Problem dank der relativen Pfade auf, hier gekennzeichnet als Punkte: ../../
Also habe ich den Makefile modifiziert, um dieses Problem zu beheben: Downloaden Sie die Datei hier und ersetzen Sie den Standard Makefile.

Ich habe die relativen Pfade so geändert, dass sie auf unser /opt/nibobeelib/ Verzeichnis verweisen.
Warum? Auf diesem Weg können wir den C Quelltext von überall aus ohne Probleme kompilieren.
Ausserdem habe ich den clean Befehl so geändert, dass sie ebenfalls .xhex Dateien löscht, da der Standard Makefile das nicht macht.
Ab diesem Zeitpunkt, sollten Sie schon den originalen Makefile mit der modifizierten Version ersetzt haben.

Also gut, bevor wir das Programm program1.c kompilieren, müssen wir erst eine .hex Datei übertragen, welche die Odometriesensoren des Roboters kalibriert.
Zuerst erstellen Sie ein neues Projekt namens calibration:

mkdir ~/Desktop/nibobee-projects/calibration/

Kopieren Sie die hex Datei aus der NIBObee Bibliothek wie folgt:

cp /opt/nibobeelib/hex/calibration.hex ../calibration

Jetzt downloaden Sie den Script hier und tun diesen in den „calibration“ Ordner oder kopieren den folgenden Code in eine Datei namens calibrate.sh,
welche in dem soeben erwähnten Ordner „calibration“ gespeichert werden sollte:

#!/bin/bash

HFUSE=0xd1
LFUSE=0xef
PROGRAMMER=usbasp
TARGET=calibration

printf '\E[31m'"\033[1mFlashing calibration program in 3 seconds\n\033[0m"
sleep 3

sudo avrdude -c $PROGRAMMER -p m16 -B 10 -U lfuse:w:$LFUSE:m -U hfuse:w:$HFUSE:m
sudo avrdude -c $PROGRAMMER -p m16 -B 2 -U flash:w:$TARGET.hex

Für diesen Script habe ich die Werte und Befehle aus dem Standard Makefile entnommen und an eine .hex Datei angepasst, weil nichts kompiliert werden muss.
Okay, schalten Sie den POWER Schalter Ihres NIBObees ein und verbinden ihn via USB mit Ihrem Computer oder Laptop,
dann gehen Sie in das „calibration“ Verzeichnis und übertragen die Kalibration Intel Hex Datei auf den Roboter mit folgenden 3 Befehlen:

cd ~/Desktop/nibobee-projects/calibration
chmod +x calibrate.sh
./calibrate.sh

Sie werden die grüne Programmierungs-LED als LED4 gekennzeichnet blinken sehen und sehen eine Reihe von Meldungen in Ihrer Konsole.
Der Script flasht die Sicherungen HFUSE, LFUSE und zu guter Letzt, den FLASH Speicher.

Falls alles gut gelaufen ist, befindet sich der NIBObee nun im Kalibrierungsmodus und Sie müssen ihn kalibrieren.
Finden Sie eine weisse oder schwarze Oberfläche, idealerweise ein Stück weisses Papier und ein Stück schwarzes Papier.
Platzieren Sie den Roboter auf eine weisse Oberfläche und drücken Sie den linken Fühler nach hinten.
Sie werden die gelben LEDs 5 mal blinken sehen.
Platzieren Sie den Roboter auf eine schwarze Oberfläche und drücken Sie den rechten Fühler nach hinten.
Sie werden die roten LEDS 5 mal blinken sehen.

Um die Kalibrierungswerte zu speichern, drücken Sie beide Fühler gleichzeitig vorwärts und alle 4 LEDs blinken 5 mal. Der NIBObee sollte nun die Kalibrierungswerte in seinem EEPROM Speicher gespeichert haben.

Sie haben dieses Tutorial fast abgeschlossen. Wir werden jetzt das erste Programm kompilieren. Gehen Sie ins program1 Verzeichnis wie folgt:

cd ~/Desktop/nibobee-projects/program1/

Tippen Sie dann make ein um den Code zu kompilieren und die .hex Datei zu übertragen:

sudo make avrdude

Wenn der Vorgang beendet ist, sollten Sie die rote LED1 konstant im selben Muster blinken sehen.
Gratulation! Sie haben erfolgreich die NIBObee Bibliothek auf Linux installiert, ein bisschen etwas über Makefile Debugging gelernt, Ihren NIBObee Roboter kalibriert und Ihr erstes Programm kompiliert als auch auf den Roboter übertragen!
